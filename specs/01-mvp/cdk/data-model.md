## Entities

### 1. Knowledge Note (CodeCommit / S3)

**Storage:** `<UID>.md` file in CodeCommit repository root (flat, no subdirectories). Replicated to S3 bucket root.

**Frontmatter Schema (YAML):**

| Field | Type | Required | Constraints | Description |
|-------|------|----------|-------------|-------------|
| `uid` | string | yes | 16-char Crockford base32 | Generated by submitKnowledge Lambda; immutable |
| `title` | string | yes | ≤255 characters, non-empty | Succinct note title |
| `status` | enum | yes | `pending` \| `active` | Lifecycle state; new notes start as `pending` |
| `author` | string | yes | ≤100 characters, non-empty | Submitter identity (trust-based in MVP) |
| `last-updated` | ISO 8601 | yes | — | Set to `submitted_at` on creation; updated by dream cycle |
| `last-linked-to` | ISO 8601 | no | — | **Not populated in MVP** (schema placeholder) |
| `last-recalled` | ISO 8601 | no | — | Updated by daily recall log processing |
| `consolidated-from-notes` | string[] | no | Format: `<filename> (<commit-hash>)` | Populated by dream cycle merge/consolidate |

**Body:** Markdown content with optional `[[UID|Title]]` wikilinks (inserted by dream cycle).

**Example File (`01H5K9QZXNM8V3PW.md`):**

```markdown
---
uid: 01H5K9QZXNM8V3PW
title: DynamoDB Global Tables require specific IAM permissions
status: pending
author: zmueller
last-updated: 2026-05-01T10:30:00Z
last-linked-to:
last-recalled:
consolidated-from-notes: []
---

## Key Insight

When configuring DynamoDB Global Tables, the IAM role needs...

## Details

- Permission `dynamodb:CreateGlobalTable` is required
- The role must have access in all target regions
```

**State Transitions:**

```
[submitKnowledge Lambda → SQS → EC2 commit] → status: pending
         │
         ▼
[Dream cycle Phase 3 on EC2]               → status: active
         │
         ▼
[Dream cycle merge on EC2]                  → file deleted (merged into another note)
```

**Validation Rules (enforced by submitKnowledge Lambda):**
- `title`: present, non-empty, ≤255 characters
- `content`: present, non-empty, ≤100,000 characters
- `author`: present, non-empty, ≤100 characters

### 2. SQS Message (Ingestion)

**Queue:** Standard SQS queue created by CDK stack

**Message Schema:**

```json
{
  "uid": "01H5K9QZXNM8V3PW",
  "title": "DynamoDB Global Tables require specific IAM permissions",
  "content": "## Key Insight\n\nWhen configuring DynamoDB Global Tables...",
  "author": "zmueller",
  "submitted_at": "2026-05-01T10:30:00Z"
}
```

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| `uid` | string | Lambda-generated | 16-char Crockford base32 |
| `title` | string | Request body (validated) | ≤255 chars |
| `content` | string | Request body (validated) | ≤100K chars |
| `author` | string | Request body (validated) | ≤100 chars |
| `submitted_at` | ISO 8601 | Lambda-generated | Current time at submission |

**Queue Configuration:**

| Property | Value | Rationale |
|----------|-------|-----------|
| Type | Standard (not FIFO) | Ordering not required |
| Visibility timeout | 5 minutes | Sufficient for EC2 batch processing |
| Message retention | 14 days | Maximum, safety net for EC2 downtime |
| Max receive count (DLQ) | 3 | Messages fail → DLQ after 3 attempts |

### 3. Recall Log (S3 Object)

**Path:** `recall-logs/<YYYY-MM-DD>/<request-id>.json`

```json
{
  "timestamp": "2026-05-01T10:30:05Z",
  "query": "how to configure replication",
  "recalled_uids": ["01H5K9QZXNM8V3PW", "01H5KABCDEF12345"]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `timestamp` | ISO 8601 | Time of recall |
| `query` | string | Original search query |
| `recalled_uids` | string[] | UIDs of notes returned to caller |

**Written by:** recallKnowledge Lambda (synchronous, best-effort)
**Read by:** EC2 server-mode CLI during daily recall log processing
**Retention:** No lifecycle rules in MVP (remains indefinitely in S3)

### 4. API Gateway Request/Response

#### submitKnowledge

**Request:** `POST /submitKnowledge`

```json
{
  "title": "Note title",
  "content": "Markdown content",
  "author": "submitter-identity"
}
```

**Success Response:** `HTTP 202`

```json
{
  "uid": "01H5K9QZXNM8V3PW",
  "request_id": "abc123-def456"
}
```

**Validation Error Response:** `HTTP 400`

```json
{
  "errors": {
    "title": "must be present and non-empty",
    "content": "must not exceed 100,000 characters"
  }
}
```

#### recallKnowledge

**Request:** `POST /recallKnowledge`

```json
{
  "query": "search query text",
  "limit": 10
}
```

**Success Response:** `HTTP 200`

```json
[
  {
    "uid": "01H5K9QZXNM8V3PW",
    "title": "Note title",
    "content": "Full Markdown content",
    "score": 0.87
  }
]
```

### 6. OpenSearch Serverless Policies

Three policy types, all managed by CDK:

#### Encryption Policy
```json
{
  "Rules": [{ "Resource": ["collection/<collection-name>"], "ResourceType": "collection" }],
  "AWSOwnedKey": true
}
```

#### Network Policy
```json
[
  {
    "Description": "VPC endpoint and Bedrock service access",
    "Rules": [
      {
        "ResourceType": "collection",
        "Resource": ["collection/<collection-name>"]
      }
    ],
    "AllowFromPublic": false,
    "SourceVPCEs": ["vpce-<endpoint-id>"],
    "SourceServices": ["bedrock.amazonaws.com"]
  }
]
```

Note (R-6): Do NOT use `AllowFromPublic: true` — it silently overrides `SourceVPCEs` and `SourceServices`, making the collection fully public. Instead, use `AllowFromPublic: false` with `SourceVPCEs` (for EC2 VPC endpoint access) + `SourceServices: ["bedrock.amazonaws.com"]` (for Bedrock internal service-to-service access). Field names are case-sensitive: `SourceVPCEs` (not `SourceVPCEndpoints`). Network policy is a JSON **array** (unlike encryption policy). Three-layer security: network origin + data access policy + IAM.

#### Data Access Policy
```json
{
  "Rules": [
    {
      "Resource": ["index/<collection-name>/*"],
      "Permission": ["aoss:ReadDocument", "aoss:WriteDocument", "aoss:CreateIndex", "aoss:DeleteIndex", "aoss:UpdateIndex", "aoss:DescribeIndex"],
      "ResourceType": "index"
    },
    {
      "Resource": ["collection/<collection-name>"],
      "Permission": ["aoss:CreateCollectionItems", "aoss:DeleteCollectionItems", "aoss:UpdateCollectionItems", "aoss:DescribeCollectionItems"],
      "ResourceType": "collection"
    }
  ],
  "Principal": [
    "<ec2-role-arn>",
    "<bedrock-kb-service-role-arn>"
  ]
}
```

  queue_url: "${SQS_QUEUE_URL}"

  endpoint: "${OPENSEARCH_ENDPOINT}"

## Entity Relationships

```
API Gateway
  ├── POST /submitKnowledge → submitKnowledge Lambda
  │     │                       ├── validates request
  │     │                       ├── generates UID + submitted_at
  │     │                       └── enqueues SQS Message
  │     │
  └── POST /recallKnowledge → recallKnowledge Lambda
        │                       ├── calls Bedrock Retrieve API
        │                       ├── optional: coverage assessment (Bedrock InvokeModel)
        │                       └── writes Recall Log to S3

SQS Queue
  └── consumed by EC2 (server-mode CLI)
        ├── creates Knowledge Note (.md) in CodeCommit
        ├── syncs to S3 (incremental)
        └── Bedrock KB indexes from S3 → OpenSearch

EC2 (server-mode CLI)
  ├── Dream Cycle
  │     ├── Phase 0: CodeCommit → S3 sync, Bedrock StartIngestionJob
  │     ├── Phase 1: OpenSearch query (status: pending)
  │     ├── Phase 2: OpenSearch query (related active notes)
  │     ├── Phase 3: Bedrock InvokeModel → apply actions → git commit
  │     └── Phase 4: S3 sync, Bedrock StartIngestionJob
  │
  └── Recall Log Processing (daily)
        ├── reads Recall Logs from S3
        └── updates last-recalled on Notes in CodeCommit

Bedrock Knowledge Base
  ├── data source: S3 bucket
  ├── vector store: OpenSearch Serverless collection
  └── queried by: recallKnowledge Lambda (Retrieve API)
```

## IAM Role Summary

| Role | Permissions | Used By |
|------|-------------|---------|
| submitKnowledge Lambda Role | `sqs:SendMessage` on queue ARN | submitKnowledge Lambda |
| recallKnowledge Lambda Role | `bedrock:Retrieve` on KB ARN, `bedrock:InvokeModel` on coverage model ARN, `s3:PutObject` on `{bucket}/recall-logs/*` | recallKnowledge Lambda |
| EC2 Instance Role | `sqs:ReceiveMessage`+`sqs:DeleteMessage` on queue ARN, `codecommit:GitPull`+`codecommit:GitPush` on repo ARN, `s3:GetObject`+`s3:PutObject`+`s3:DeleteObject`+`s3:ListBucket` on bucket ARN, `s3:GetObject` on CLI binary S3 URI, `aoss:APIAccessAll` on collection ARN, `bedrock:InvokeModel` on consolidation model ARN, `bedrock-agent:StartIngestionJob`+`bedrock-agent:GetIngestionJob` on KB/data source, `ssm:*` (Session Manager) | EC2 instance |
| Bedrock KB Service Role | `s3:GetObject`+`s3:ListBucket` on bucket ARN, `aoss:APIAccessAll` on collection ARN, **`bedrock:InvokeModel` on embedding model ARN** (R-2) | Bedrock KB data source sync (ingestion + retrieval) |
| Index Creation Lambda Role | `aoss:APIAccessAll` on collection ARN (also listed in AOSS data access policy) | Custom resource Lambda for OpenSearch index pre-creation (R-2) |
